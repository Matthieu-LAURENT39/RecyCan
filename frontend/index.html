About the return process

In a real deployment, the whole return flow would be handled by a specialized return machine.
The user would place the bottle in the machine in a way where the user can no longer retrieve it. It would then scan the barcode, and validate that the product is a match (correct weight and/or dimensions), before finally authorizing the refund. This is similar to how existing collection sheme machines work.
Only the address associated with those machines would be marked as "authorized" to process returns in the smart contract, and would be the only ones able to trigger refunds.
For demo purposes, this whole process is simulated by this web interface.
